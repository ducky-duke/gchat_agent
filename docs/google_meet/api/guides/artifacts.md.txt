This document explains how to get details about the different meeting artifacts
generated by a conference using the Google Meet REST API.

An [artifact](https://developers.google.com/workspace/meet/api/guides/overview#artifact) is a file or data
generated by Google Meet in response to a
[conference](https://developers.google.com/workspace/meet/api/guides/overview#conference). This includes
video recordings and other artifacts, such as transcripts.

To produce artifacts, participants must configure them in Meet
before the conference ends. For more information, see [Manage auto
artifacts](https://developers.google.com/workspace/meet/api/guides/meeting-spaces-configuration#auto-artifacts).

Transcripts operate independently of recordings and you don't have to record the
meeting to generate a transcript. When all participants leave the meeting, the
transcript stops automatically. For more information, see [Record a video
meeting](https://support.google.com/meet/answer/9308681) and [Use Transcripts
with Google Meet](https://support.google.com/meet/answer/12849897).

## Retrieve artifacts

If you're a meeting space owner or participant, you can call the `get` and
`list` methods on the `recordings`, `transcripts`, `transcripts.entries`, and
`smart notes` resources to retrieve artifacts. For more information, see the
following sections.

Authenticating and authorizing with [user
credentials](https://developers.google.com/workspace/meet/api/guides/authenticate-authorize) lets
Meet apps access user data and perform operations on the
authenticated user's behalf. Authenticating with [domain-wide
delegation](https://developers.google.com/workspace/meet/api/guides/authenticate-authorize#domain-wide-delegation)
lets you authorize an application's service account to access your users' data
without requiring each user to give consent.

## Artifact retention

After a conference ends, Meet saves recordings and transcripts to
the meeting organizer's Google Drive. By default, Meet
artifacts are retained according to Drive rules.

Transcript entries provided by the Meet REST API are deleted 30 days after
the conference ends.

You also can manage the retention of Meet artifacts separately
using Meet-specific retention rules in Google Vault. For more
information, see [Retain Google Meet data with
Vault](https://support.google.com/vault/answer/7682297).

## Recordings

The following sections detail how to get information about recordings in a
conference record.

Meet has access to the recording ID after the recording file is
generated. It's possible a user might delete the recording file from
Google Drive, yet the unique name is still returned in Meet.

The
[`recordings`](https://developers.google.com/workspace/meet/api/reference/rest/v2/conferenceRecords.recordings)
resource includes both the
[`DriveDestination`](https://developers.google.com/workspace/meet/api/reference/rest/v2/conferenceRecords.recordings#drivedestination)
and the
[`State`](https://developers.google.com/workspace/meet/api/reference/rest/v2/conferenceRecords.recordings#state)
objects.

The `DriveDestination` object holds the export location in Drive
where the recording is saved as an MP4 file. To download the recording or to
play it back in a browser, use the object's
[`exportUri`](https://developers.google.com/workspace/meet/api/reference/rest/v2/conferenceRecords.recordings#DriveDestination.FIELDS.export_uri)
field value. If you're familiar with the Google Drive API, the
[`file`](https://developers.google.com/workspace/meet/api/reference/rest/v2/conferenceRecords.recordings#DriveDestination.FIELDS.file)
field value corresponds to the
[`id`](https://developers.google.com/workspace/drive/api/reference/rest/v3/files#File.FIELDS.id) field in the
[`files`](https://developers.google.com/workspace/drive/api/reference/rest/v3/files) resource. For more
information, see [Download and export
files](https://developers.google.com/workspace/drive/api/guides/manage-downloads).

The `State` object holds the status of the recording session. This lets you
query if the recording has started, if the recording has ended but the recording
file isn't ready, or if the recording file is generated and ready to download.

## Transcript entries

The following sections detail how to get each participant's speech as transcript
entries during a conference transcript session.

The transcript entry contains transcribed text of the participant's voice, up to
a maximum of 10,000 words. The spoken text `languageCode` (for example, `en-US`)
is also included as [IETF BCP 47 syntax](https://tools.ietf.org/html/bcp47).

Each transcript entry is connected to a `participant` name in the conference. To
retrieve participant details, see [Get details about a
participant](https://developers.google.com/workspace/meet/api/guides/participants#get-participant).

Transcript entry data is available for 30 days after the conference ends.

Note that the transcript entries returned by the Meet REST API might not
match the transcription found in the Docs transcript file. This
occurs when the transcript file is modified after generation.

## Smart notes

The following sections detail how to get information about conference smart
notes in a conference record.

Meet has access to the smart note ID once the smart note data is
generated. It's possible a user might delete the smart notes file from
Drive, yet the unique name is still returned in
Meet.

The
[`smartNotes`](https://developers.google.com/workspace/meet/api/reference/rest/v2/conferenceRecords.smartNotes)
resource includes the
[`State`](https://developers.google.com/workspace/meet/api/reference/rest/v2/conferenceRecords.smartNotes#state)
object. This object holds the status of the smart notes session. This lets you
query if the smart notes session has started, if the smart notes session has
ended but the smart notes file isn't ready, or if the smart notes file is
generated and ready to download.

The `smartNotes` resource also includes the
[`DocsDestination`](https://developers.google.com/workspace/meet/api/reference/rest/v2/DocsDestination)
object type. This object holds the export location in Drive where
the Google Docs smart notes file is saved. To fetch the content or to browse
the smart notes in a browser, use the object's
[`exportUri`](https://developers.google.com/workspace/meet/api/reference/rest/v2/DocsDestination#FIELDS.export_uri)
field value. If you're familiar with the Google Docs API, the
[`document`](https://developers.google.com/workspace/meet/api/reference/rest/v2/DocsDestination#FIELDS.document)
field value corresponds to the
[`documentId`](https://developers.google.com/workspace/docs/api/reference/rest/v1/documents#Document.FIELDS.document_id)
in the [`documents`](https://developers.google.com/workspace/docs/api/reference/rest/v1/documents) resource.

You can manage smart notes using the [`get`](https://developers.google.com/workspace/meet/api/guides/artifacts#get-smart-notes) and
[`list`](https://developers.google.com/workspace/meet/api/guides/artifacts#list-smart-notes) methods.

## Customize pagination

Pass the following query parameters to customize pagination of recordings,
transcripts, and transcript entries:

- `pageSize`: The maximum number of items to return. The service might return
  fewer than this value. If unspecified, at most 10 items are returned. The
  maximum value is 100; values more than 100 are automatically changed to 100.

- `pageToken`: A page token, received from a previous list call. Provide this
  token to retrieve the subsequent page.

> [!NOTE]
> **Note:** Unlike `conferenceRecords`, `participants`, and `participantSessions` resources, you cannot filter query results for `recordings`, `transcripts`, and `transcripts.entries` resources.
